Notes

  • call() and apply() methods can send parameters to functions by connecting a method to an object
  • Memoization provides result caching
  • Immediately Invoked Function Expressions (IIFE) or iffy is invoked as soon as it is defined.
  • Safe use of strict mode is to put your code inside an IIFE
  • Functions that define and rewirte themselves - this is nuts
  • Recursive Functions exist in JS
  • Callbacks facilitate event-driven asynchronous programming
  • Promises wait for an asynchronous operation
  • .then() for the promise outcome
  • .catch() if the operation fails
  • Functions can return functions
  • Closures - this is confusing
  • A closure is formed with an innter function is returned by an outer function
  • Generators produce iterators that maintain the state of the value
  • Functional Programming is not object oriented
  • Functional Programming uses pure function
  • Pure functions:
    1. return value should only depend on arguments provided
    2. No side effects - doesn't change data anywhere else in the program
    3. Given the same arguements, the pure function will always return the same results
  • Higher-order functions are functions that accept another function as an argument, or return another function as a result


  • AJAX allows pages to be updated without reloading the whole page
  • Using fetch is so much easier
  • fetch starts a promise and has a response interface that is returned when the promise is fulfilled
  • fetch can use redirect(), text() or json()
  • Request object is sent as an argument with fetch
  • Don't forget Headers
